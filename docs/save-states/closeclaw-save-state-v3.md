## 12. Arquitectura Operativa

### 12.1 Repositorio (`err0r403/closeclaw`)
El cerebro persistente de CloseClaw. Todo estado, plan y decisión vive aquí.

- **Git worktree**: `master/` es la branch principal. Branches adicionales para features/experimentos.
- **`docs/`**: Save states, chat logs, specs de productos, documentación operativa.
- **`CLAUDE.md`**: Contexto para cualquier instancia de Claude Code que opere en el repo.

### 12.2 Formato de Documentación
Adoptamos estructura numerada y modular (inspirada en specs técnicas):

| Tipo de documento | Propósito | Ubicación |
|-------------------|-----------|-----------|
| Save State | Snapshot completo del agente en un punto del tiempo | `docs/closeclaw-save-state-vN.md` |
| Chat Log | Registro histórico de conversaciones fundacionales | `docs/chats/*.log` |
| Product Spec | Especificación detallada de un producto/servicio | `docs/specs/` (por crear) |
| Playbook | Guías operativas (cómo escribir propuestas, pricing, etc.) | `docs/playbooks/` (por crear) |

### 12.3 Flujo de Trabajo
1. **Continuidad**: Toda nueva sesión comienza leyendo el save state más reciente + CLAUDE.md
2. **Persistencia**: Al cerrar una sesión productiva, generar nuevo save state
3. **Branching**: Usar worktrees para explorar ideas sin contaminar master
4. **Decisiones**: Se registran en el save state con contexto y prioridad
